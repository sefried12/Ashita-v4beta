            <settings>

                        <!-- All setting nodes are optional.  If they are not specified, or an invalid value is used, the default value(listed in white here) will be used. -->

                        <packetws>false</packetws> <!-- When enabled, weaponskills will be sent as packets.  This allows you to WS disengaged.  This does not work with <st> type targeting. -->

                        <gearcheck>true</gearcheck> <!-- When enabled, shorthand will block Honor March, Impact, and Dispelga packets from going out if the correct gear is not currently in an equippable bag. -->

                        <!--Impact, Honor, and Dispelga switches will modify the client to show that you always know the corresponding spell.

                        Please note that this will not allow you to cast the spell on it’s own.  It is simply to streamline everything.

                        You will still need to configure ashitacast or some sort of equip macro to ensure the corresponding equipment is put on in precast and midcast. -->

                        <impact>false</impact>

                        <honor>false</honor>

                        <dispelga>false</dispelga>

                        <debug>false</debug> <!-- When enabled, all typed commands will be printed to Ashita’s debug log.  You must have Ashita logging set to debug to see this. -->

            </settings>

 

            <!-- Ability, Item, Spell, and Weaponskill tags can be used to further customize inputs. You should comment the name for ease of use, as I have done.

Note that you can use the command ‘/shh export’ to print a xml full of pregenerated ability/spell/weaponskill tags to ashita/config/shorthand/settings-empty.xml for copy/paste usage.

Item tags simply use the item’s ID, which you can find on FFXIAH.  Note that you can only alias usable or enchanted items for actual use, this is not used to trade items to players or NPCs. -->

                        <command>/castcure</command> <!--A command tag replaces the entire body of a command.  So, you can use ‘/castcure PlayerName’ in this case.-->

                        <name>heal</name> <!--A name tag adds an additional name to the internal registry.  So, you can use ‘//heal PlayerName’ or ‘/ma heal PlayerName’ in this case.-->

                        <!--You can use any number of either type of tag to add additional ways to cast a spell.-->

            </spell>
